ZAP Automation Configuration

Environment Settings

Defines the core environment settings including contexts and authentication.

env:
  contexts:
    - name: context 1
      urls:
        - "http://example.com"

Authentication and Verification

Details on authentication methods and verification mechanisms to ensure validity of session states.

authentication:
  method: form
  parameters:
    loginPageUrl: "http://example.com/login"
    loginRequestUrl: "http://example.com/session"
    loginRequestBody: "username={username}&password={password}"
verification:
  method: response
  loggedInRegex: "Log Out"

Session Management and Technology Settings

Configures session handling and specifies technologies to include or exclude during scans.

sessionManagement:
  method: cookie
technology:
  include:
    - "SQL Injection"
    - "XSS"

Job Configurations

Sets defines jobs like requestor and alert filters with specific parameters.

- type: alertFilter
  parameters:
    deleteGlobalAlerts: true
  alertFilters:
    - ruleId: 10010
      newRisk: High
      context: User Authentication
      url: "http://example.com/login"
      urlRegex: false
      parameter: "sessionToken"
      parameterRegex: true
      attack: "SQL Injection"
      attackRegex: false
      evidence: "Unauthorized"
      evidenceRegex: true
    - ruleId: 20020
      newRisk: Info
      context: Public API
      url: "http://api.example.com/v1/"
      urlRegex: true
- type: requestor
  parameters:
    user: adminUser
  requests:
    - url: "http://example.com/api/data"
      method: POST
      httpVersion: HTTP/2
      headers:
        - "Content-Type:application/json"
        - "Authorization:Bearer {token}"
      data: '{"query":"fetch"}'
      responseCode: 200
    - url: "http://example.com/api/update"
      method: PUT
      httpVersion: HTTP/1.1
      headers:
        - "Content-Type:application/json"
        - "X-Custom-Header:1234"
      data: '{"update":"true"}'
      responseCode: 202
