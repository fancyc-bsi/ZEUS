Configuration Examples

Zeus uses two configuration files, one for severity and the other for verification. Modifications can be made as follows:

Severity Configuration

Modify any finding's severity by changing the value in the configuration file. If an entry does not exist for a target finding, create it as shown in the scan results.

Verification Configuration

The verification configuration file contains several parts that can be adjusted according to user needs:

  • Name: Identifier of the finding.
  • WSTG Category: The corresponding category from the Web Security Testing Guide.
  • Matcher: Defines how findings are matched.
  • Screenshot: Specifies if a screenshot is required and the type of screenshot.

Matcher Types Explained

  • Reverse Matcher: Checks for the absence of specified headers or patterns. Used to identify security headers that are not set.
  • Regex Matcher: Uses regular expressions to match content in response headers or body. Highly versatile and powerful for complex pattern matching.
  • Compound Matcher: Combines multiple conditions using logical operators (AND, OR). Allows for complex logical constructs in detection scenarios.
  • Status Matcher: Looks for specific HTTP status codes in responses. Useful for detecting specific server errors or redirect patterns.
  • Word Matcher: Matches whole words in the content. Simpler and less resource-intensive than regex for basic tasks.

Screenshot Types Explained

  • File: Takes a screenshot of the rendered HTML, typically used when headers are the target type. This method captures the visual output of header manipulations directly from the browser.
  • Web: Goes directly to the URL location on the web while respecting the proxy setup used during the initial execution. This is ideal for capturing real-world web scenarios as they appear to the end-user.

TOML Configuration (src/config/severity_config.toml)

["Content Security Policy (CSP) Header Not Set"]
severity = "Low"

["X-Content-Type-Options Header Missing"]
severity = "Low"

JSON Configuration (src/config/verification_config.json)

{
    "findings": [
        {
            "name": "Content Security Policy (CSP) Header Not Set",
            "wstg_category": "WSTG-CONF-06",
            "matcher": {
                "type": "reverse",
                "headers": ["Content-Security-Policy"]
            },
            "screenshot_required": true,
            "screenshot_type": "file"
        },
        {
            "name": "Server Leaks Version Information via \"Server\" HTTP Response Header Field",
            "wstg_category": "WSTG-INFO-02",
            "matcher": {
                "type": "regex",
                "scope": "header",
                "pattern": ".*(Apache|nginx|IIS)/\\d+\\.\\d+(\\.\\d+)?"
            },
            "screenshot_required": true,
            "screenshot_type": "file"
        },
        {
            "name": ".htaccess Information Leak",
            "wstg_category": "WSTG-INFO-03",
            "matcher": {
              "type": "compound",
              "logic": "or",
              "conditions": [
                {
                  "type": "status",
                  "status_code": 200
                },
                {
                  "type": "regex",
                  "scope": "body",
                  "pattern": "AddType|AddHandler|AuthType|Require"
                }
              ]
            },
            "screenshot_required": true,
            "screenshot_type": "web"
          }
    ]
}
